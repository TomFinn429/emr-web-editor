# pTemplateMake 高保真 MVP 还原设计

## 背景

目标页面为 `http://10.0.15.11:8766/dist_test/index.html#/pTemplateMake` 的模板制作工具。经 MCP Playwright 查阅，目标页是以 DCWriter 为核心的模板制作工作台，包含主模块导航、模板树、模板多页签、Ribbon 工具条、元数据/片段模板辅助区、DCWriter 编辑画布、模板属性和元素属性面板。

当前 `emr-web-editor-mvp` 已具备相似的工作台骨架和大量可复用能力：模板树、多页签、保存/另存为/上传 mock、元数据插入、片段模板插入、元素属性读写、统一弹窗、WriterControl 适配和命令注册表。本设计采用方案 A：在现有工作台上高保真补齐目标页功能面，而不是重写页面或优先接真实后端。

## 目标

第一期目标是让 MVP 在视觉结构、主要入口、状态表达和核心交互上接近目标 `pTemplateMake` 页面。

- 对齐三栏工作台：左侧模板树，中左辅助面板，中间编辑区，右侧属性面板。
- 对齐目标页的工具入口：文件、编辑、格式、段落、插入、设计和高级相关命令。
- 对齐模板树能力：范围筛选、搜索、新建文件夹、新建模板、编辑名称、删除、状态标签。
- 对齐属性面板：模板属性字段补齐，元素属性按输入域/单元格/表格行/表格切换。
- 保留当前 DCWriter/WriterControl 渲染、保存、打印、插入和属性读写适配路径。
- 保留 service 和 adapter 边界，使真实 API 后续可替换 mock 数据。

## 非目标

第一期不做完整后台系统复刻。

- 不接完整权限系统和登录态。
- 不实现完整上传审批、版本对比、回滚和业务发布流。
- 不复刻目标系统所有主菜单模块，只在模板制作页提供必要视觉入口或明确不可用反馈。
- 不重写 DCWriter XML 渲染和编辑引擎。
- 不在 Vue 组件中直接访问 WriterControl 私有结构。
- 不绕过 `commandRegistry` 在按钮中硬编码 WriterControl 命令。

## 目标页功能观察

目标页具备以下主要区域：

- 顶部壳层：面包屑“模板制作”、显示模式、全屏、重载、账号入口。
- 左侧主导航：仪表板、模板制作、数据元管理、字典库管理、片段模板库管理、页眉页脚库管理、模板管理、元数据管理、值域管理、数据源管理、模板类型管理等。
- 模板列表：范围选择“全局”、关键字检索、树形分类、模板文件、待上传状态、新增文件夹和新增模板。
- 中间工作区：打开模板页签显示 dirty 星号，编辑区加载 DCWriter 文档。
- Ribbon 工具条：保存、上传、取消上传、撤销、重做、剪切、复制、粘贴、字体、字号、加粗、斜体、下划线、删除线、文字颜色、背景颜色、对齐、特殊字符、图片、页眉页脚、设计模式、刷新文档、XML 源码。
- 辅助面板：元数据和片段模板两个页签，支持刷新、过滤和树形选择；源代码中可见元数据拖拽插入、片段模板 `InsertXML` 插入路径。
- 右侧属性：模板属性和元素属性。模板属性包含名称、分类、类型、打印方式、是否允许重复、签署级别、使用科室、作者、更新者和更新时间。元素属性包含元素类型、更新按钮、编号、名称、背景文字、边框、对齐、固定宽度、绑定数据源、输入/输出格式、只读、校验、表达式、颜色、自定义属性和打印可见性。

目标页的相关接口包括 `template/org/list`、`template/xml`、`metadata/category/org/list`、`template/block/category/org/list`、`template/hf/category/org/list`、`pic/category/org/list`、`special/char/category/org/list`。当前查阅时接口返回 token 过期，因此设计以已加载 UI、DOM、静态 JS bundle 和现有 MVP 能力为依据。

## 推荐方案

采用“增量对齐现有工作台”。

`EditorShell.vue` 继续作为组合层，负责加载工作台数据、维护当前模板、接收子组件事件、分发 WriterControl 命令和刷新状态。目标页还原通过扩展现有组件和 service 数据结构完成，不新建平行工作台。

备选方案及取舍：

- 新建仿真壳层再迁移能力：视觉更容易从空白处还原，但会制造两套工作台状态和事件链路，后续维护风险高。
- 先抽配置化工作台再还原 UI：长期灵活，但第一期交付慢，并且当前 `commandRegistry` 和属性 service 已经提供足够边界。

## 架构设计

### 组合层

`EditorShell.vue` 保持薄组合层定位，但需要降低继续膨胀的风险。第一期允许它继续承接跨区域事件，但新增逻辑应优先沉入 service、adapter 或小型 composable。

职责：

- 加载和刷新 `templateWorkbenchService` 数据。
- 管理当前模板、打开页签、dirty 状态和状态文案。
- 分发模板树、工具条、辅助面板、属性面板事件。
- 通过 `writerControlAdapter` 和 `writerElementAdapter` 调用 DCWriter。

### 模板树

`TemplateTreePanel.vue` 负责目标页左侧模板树体验。

新增或强化：

- 范围选择：全局、全院、科室、个人。
- 搜索框文案与目标页对齐。
- 顶部更多菜单：新增文件夹、新增模板。
- 节点更多菜单：新增文件夹、新增模板、编辑名称、删除文件夹、删除文件。
- 状态标签：待上传、未上传、已上传、上传中、已取消、上传失败。

业务数据仍来自 `templateWorkbenchService.ts`，组件只消费 props 并上抛事件。

### Ribbon 工具条

`WorkbenchTopMenu.vue` 和 `commandRegistry.ts` 继续作为工具条来源。

第一期不追求目标页所有原生命令 1:1 可用，而是对齐入口和反馈：

- 已有可用命令直接保留真实执行。
- 已有 adapter 但底层不稳定的命令保留明确状态反馈。
- 目标页有但 MVP 暂不支持的入口以禁用状态或明确不可用反馈呈现。

命令分组建议：

- 文件：保存、另存为、下载 XML、上传、批量上传、取消上传、历史版本、打印、打印预览。
- 编辑：撤销、重做、剪切、复制、粘贴、全选。
- 格式：字体、字号、加粗、斜体、下划线、删除线、文字颜色、背景颜色。
- 段落：左对齐、居中、右对齐、分散对齐。
- 常规/插入：输入域、日期时间、单选框、复选框、页眉页脚、特殊字符、图片、条码、二维码、分页符。
- 设计/高级：设计模式、刷新文档、XML 源码、模板属性、元素属性、版本或对比禁用入口。

### 辅助面板

`WorkbenchAssistPanel.vue` 继续承载元数据和片段模板。

元数据：

- 支持刷新、搜索、树形展示、选择。
- 保留现有“插入为输入域”和“绑定当前元素”闭环。
- 目标页的拖拽插入作为第一期增强项：若实现成本过高，可先通过按钮插入达成可验收闭环。

片段模板：

- 支持刷新、搜索、树形展示、选择和插入。
- 继续通过 `writerElementAdapter.insertFragmentTemplate()` 调用 `InsertXmlString`。
- 片段 XML 继续由 `fragmentTemplateService.ts` 提供，避免组件拼 XML。

### 属性面板

`PropertyInspectorPanel.vue` 继续作为右侧外壳，承载模板属性和元素属性。

模板属性：

- 字段对齐目标页：模板名称、模板分类、模板类型、打印方式、是否允许重复、模板签署级别、使用科室、作者、更新者、最后更新时间。
- 第一期允许部分字段只读或 mock 下拉。
- 属性保存入口保留在 service 边界。

元素属性：

- 支持元素类型切换：输入域、单元格、表格行、表格。
- 输入域字段优先对齐目标页可见字段：编号、名称、背景文字、提示信息、标签文本、单位文本、边框、内容对齐方式、固定宽度、焦点快捷键、激活模式、高亮度状态、隐藏、数据源名称、绑定路径、Text 绑定路径、输入格式、输出格式、只读、允许删除、允许键盘编辑、最大内容长度、加密显示、校验、仅可输入字符、计算表达式、可见性表达式、打印可见性表达式、文字颜色、背景文字颜色、自定义属性、打印可见性。
- 字段渲染应采用 schema 或集中配置，避免在 SFC 中堆大量重复表单。
- 读写仍通过 `useElementInspector` 和 `writerElementAdapter`。

## 数据流

页面启动：

1. `EditorShell` 调用 `fetchTemplateWorkbenchData()`。
2. service 返回模板树、范围、元数据树、片段模板树、模板属性、元素属性、历史版本和打开页签。
3. Shell 将数据拆给模板树、辅助面板、编辑区和属性面板。

打开模板：

1. 用户点击模板树节点。
2. Shell 调用 `openTemplateContent(id)`。
3. `useDocumentSession` 加载模板 XML。
4. `WorkbenchEditorArea` 交给 DCWriter 外部渲染。
5. Shell 刷新属性面板、页签和状态栏。

保存模板：

1. 工具条触发 `save`。
2. Shell 通过 `writerControlAdapter.saveXml()` 获取 XML。
3. `documentSaveService` 尝试真实保存。
4. 若真实保存失败，保留当前第四阶段策略，明确提示并保存到 `templateWorkbenchService` mock 模板。
5. 更新 dirty、版本、上传状态和状态文案。

元数据和片段插入：

1. 辅助面板发出选择或插入事件。
2. Shell 将元数据转为输入域属性，或将片段 XML 交给 adapter。
3. adapter 调用 WriterControl。
4. 成功后更新状态文案，失败时给出明确不可用/失败原因。

元素属性读写：

1. 用户切到元素属性并点击更新，或后续接入选择事件后自动读取。
2. `useElementInspector` 通过 adapter 读取当前元素。
3. 用户修改字段后，patch 经 adapter 写回 WriterControl。
4. 面板展示同步结果。

## 错误处理

- API 失败：保留明确错误消息，mock 数据不静默覆盖真实错误。
- WriterControl 命令缺失：按钮可见但反馈“不支持”或禁用，并在状态栏展示原因。
- 保存失败：真实接口失败时继续 mock 兜底，但必须提示真实保存不可用。
- 未保存切换：复用统一 `WorkbenchDialog`，避免浏览器原生确认框。
- 插入失败：元数据、片段、图片、特殊字符都需要给出失败提示，不吞异常。

## 测试与验收

单元测试：

- `templateWorkbenchService`：范围筛选、树节点 CRUD、状态流转、属性补齐。
- `commandRegistry`：目标页新增入口、禁用状态、payload 生成。
- 元数据/片段 service：搜索、树形数据、插入 XML 输出。
- 元素属性 service/schema：字段映射和 patch 生成。
- dialog/composable：未保存切换和删除确认。

构建验证：

- `cd frontend && npm test`
- `cd frontend && npm run build`

Playwright 验收：

- 页面打开后呈现目标页式工作台布局。
- 模板树显示范围选择、搜索、分类、模板、状态标签和更多菜单。
- 点击模板后打开页签，dirty 星号和状态文案可见。
- 工具条核心入口可见，已支持命令可执行，不支持命令有明确反馈。
- 元数据可搜索、选择、插入或绑定。
- 片段模板可搜索、选择、插入。
- 模板属性字段完整显示。
- 元素属性可读取和写回当前元素。
- 保存失败兜底和下载 XML 不破坏文档结构。

## 实施顺序建议

1. 先对齐布局和壳层视觉，确认四栏比例、页签、工具条和状态文案。
2. 扩展 `commandRegistry` 与 `WorkbenchTopMenu`，补齐目标页入口和明确不可用反馈。
3. 强化模板树范围选择、节点菜单和状态标签。
4. 补齐模板属性和元素属性 schema，保持读写路径不变。
5. 强化元数据和片段模板体验。
6. 补单元测试。
7. 跑构建和 Playwright 验收。

## 设计约束

- 继续使用 Vue 3、Composition API、`<script setup lang="ts">`。
- 组件采用 props down、events up。
- 复杂 UI 拆到小组件或 composable，避免继续扩大 `EditorShell.vue`。
- 业务数据集中在 service，组件不硬编码模板、元数据、字典、值域、片段模板数据。
- WriterControl 访问集中在 adapter/composable。
- 第一期以高保真 MVP 为目标，真实 API 接入留给后续阶段。
