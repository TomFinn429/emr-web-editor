# 电子病历 Web 编辑器月度成果展示路线图

## 目标

本路线图用于把研发成果拆成可控的月度汇报版本。每个月只展示已经定义好边界的成果，允许存在已知缺陷，但缺陷必须被记录、可解释，并且不影响当月汇报主线。

## 展示原则

- 默认启动入口为第 2 月 XML 导入闭环：`http://127.0.0.1:5174/?stage=month-2-import-preview`。
- 完整编辑器入口仍保留：`?stage=full-editor`。
- 月度汇报入口使用固定阶段参数，例如：`?stage=month-2-import-preview`。
- 每个阶段必须有演示动作、成功口径、已知问题和下月目标。
- 未进入本月验收范围的能力不在汇报入口暴露，避免被误解为已经承诺完成。

## 月度阶段规划

| 阶段 | 汇报主题 | 展示范围 | 可接受缺陷 | 不展示内容 |
| --- | --- | --- | --- | --- |
| 第 2 月 | XML 导入闭环 | 用户可导入 XML 并进入渲染链路 | 部分导入 XML 版式错乱，失败时给明确提示 | 全量 XML 兼容承诺、模板制作能力 |
| 第 3 月 | 渲染兼容性提升 | 固定样例扩展到 8-10 个，记录每个样例评分 | 复杂控件、图片、特殊表格仍可异常 | 编辑与保存 |
| 第 4 月 | 预览工作台雏形 | 模板列表、打开模板、预览、缩放 | 模板数据可为 mock，状态不持久化 | 高级编辑、元数据、片段模板 |
| 第 5 月 | 基础编辑能力 | 基础文本命令、少量 WriterControl 命令 | 不支持命令必须有明确提示 | 复杂控件编辑 |
| 第 6 月 | 保存与导出 | 保存当前 XML、下载 XML、保留结构 | 后端保存不可用时允许 mock 降级保存 | 真实模板库闭环 |
| 第 7 月 | 模板制作初版 | 模板树、多页签、保存、另存为、上传状态 mock | 上传审批和历史版本可继续模拟 | 真实后台管理 |
| 第 8 月 | 高级能力试点 | 元数据插入、片段模板插入、属性面板部分联动 | 只支持已确认 API 和控件 | 全量 DCWriter 能力承诺 |

## 第 2 月展示入口

启动前端后访问：

```text
http://127.0.0.1:5174/?stage=month-2-import-preview
```

也可以通过环境变量进入：

```text
VITE_PRESENTATION_STAGE=month-2-import-preview
```

完整编辑器仍可通过以下参数访问：

```text
http://127.0.0.1:5174/?stage=full-editor
```

## 第 2 月导入闭环验收口径

| 演示动作 | 本月成功口径 | 已知边界 |
| --- | --- | --- |
| 选择本地 XML 文件 | 文件通过基础格式校验，页面状态显示当前文件名 | 空文件、超大文件、非 XML 内容会被拒绝 |
| 进入渲染链路 | 校验通过后调用现有 `CanvasPreview.vue` 和 `useCanvasRenderer.ts` | 复杂表格、页眉页脚、控件状态和分页可能错乱 |
| 展示导入结果 | 外部渲染可用时显示外部渲染状态，不可用时降级为结构化预览 | 本阶段不承诺任意 XML 版式完全正确 |
| 调整缩放 | 用户可以放大、缩小和重置缩放 | 不开放编辑、保存、上传和模板制作 |

## 每月汇报包格式

每个月汇报前产出以下内容：

- 展示 URL：明确阶段参数。
- 演示动作：列出本月允许演示的用户操作。
- 成功口径：说明哪些视觉和流程效果算完成。
- 已知问题：用截图或文字记录，不在现场临时解释。
- 本月新增：对比上月新增的能力。
- 下月目标：只列 2-3 个确定目标。

## 工程约束

- 月度展示入口不得直接扩大 `EditorShell.vue` 职责。
- 导入展示应复用 `useDocumentImport.ts`、`CanvasPreview.vue` 和 `useCanvasRenderer.ts`。
- 阶段配置集中在 `frontend/src/presentation/stageProfiles.ts`。
- 后续新增阶段时先补 `stageProfiles.test.ts`，再实现入口。
- 未验证的 WriterControl 命令不进入月度汇报入口。
