主流程说明文档

电子病历 Web 编辑器从进入工作台到保存输出的完整链路

本文基于当前仓库代码梳理，目标是让产品、研发和交付同事快速理解：页面如何进入编辑器、模板和本地 XML 如何加载、编辑行为如何标记未保存、保存与导出如何落到接口或本地工作台状态，以及外部 DCWriter 渲染不可用时如何降级。

生成位置：根目录 / main-flow-docs 范围：前端工作台 + 后端 API 边界 更新日期：2026-06-02

一、总体结论

当前项目的核心不是传统多页面路由，而是一个 Vue 工作台壳层。App.vue 根据演示阶段决定展示月度演示页或进入 EditorShell。进入主编辑器后，EditorShell 统一协调模板树、辅助面板、编辑区、属性面板、状态栏和对话框。

1. 入口判断App.vue 读取 query/env stage，普通主编辑流程进入 EditorShell。
2. 工作台初始化挂载后调用 fetchTemplateWorkbenchData 拉取模板树、元数据、片段模板、属性和页签。
3. 打开文档用户选择模板或导入本地 XML，写入 useDocumentSession 的当前文档会话。
4. 渲染编辑CanvasPreview 优先尝试 DCWriter 外部渲染，失败后使用结构化 Canvas 预览。
5. 操作与标脏菜单命令、内容变化、元数据绑定、片段插入都会通过会话或模板状态标记未保存。
6. 保存输出保存先从 WriterControl 取 XML 并校验，再 POST 后端；导出则生成 XML 下载。
代码中没有发现独立 LLM/AI 服务主链路。WorkbenchAssistPanel 是“编辑辅助面板”，职责是元数据与片段模板，不等同于 AI 对话或自动生成。

二、入口与初始化流程

页面入口
  1. frontend/src/App.vue 调用 resolvePresentationStage。
  2. 当 stage 为 xml-import 时进入演示页 XmlImportMilestone。
  3. 其他情况进入主编辑工作台 EditorShell。
工作台挂载
  1. EditorShell 在 onMounted 中执行 loadWorkbenchData()。
  2. loadWorkbenchData() 调用 fetchTemplateWorkbenchData(currentTemplateId)。
  3. 返回数据驱动模板树、元数据树、片段模板树、模板属性、历史版本和打开页签。
数据来源当前实现特点
模板列表与模板内容templateService.ts 调用 /api/templates 和 /api/templates/{id}开发环境先由 devLocalTemplateApiPlugin 从 D:/XML/notes 读取 XML；未被插件处理的 /api 请求再代理到后端。
工作台状态templateWorkbenchService.ts维护内存态模板树、页签、历史版本、上传状态和模板属性，是当前 MVP 的工作台状态中心。
后端模板与保存 APIbackend/Program.cs提供 GET /api/templates、GET /api/templates/{id}、POST /api/documents/save，保存结果进入内存 SavedDocumentStore。

三、打开文档的两条路径

路径 A：从模板树打开
  1. 用户点击模板树节点，触发 handleTemplateTreeSelect(node)。
  2. 若当前文档已标脏，canReplaceCurrentDocumentAsync 弹出确认。
  3. handleTemplateSelect(id) 调用 openTemplateContent(id)。
  4. openTemplateContent 读取模板 XML、确保页签打开，并更新 active template。
  5. session.loadTemplate(template) 写入当前会话，保存状态设为 saved。
  6. refreshWorkbenchData(id) 刷新右侧属性、历史版本和打开页签。
路径 B：导入本地 XML
  1. 顶部菜单触发 handleLocalImport(file)。
  2. useDocumentImport.importFile 检查文件大小、XML 格式和解析结果。
  3. 导入时会对演示敏感标识做脱敏，并生成本地文档 id。
  4. session.loadLocalDocument(imported) 写入当前会话，来源标记为 local。
  5. 本地导入不绑定模板 id，因此保存失败时不会走模板 mock 降级保存。

四、渲染与编辑链路

当前文档进入会话后，会转换为预览输入并传给编辑区。真实渲染由 useCanvasRenderer 决策：优先加载 DCWriter 外部渲染资源，失败时自动回退到前端 Canvas 结构化预览。

阶段关键函数或组件说明
预览对象转换toPreviewDocument(session.document.value)把 EditorDocument 转成 ImportedDocument，交给 WorkbenchEditorArea。
渲染触发CanvasPreview watch props.document文档为空时清空画布；文档变化时调用 renderDocument(host, document)。
外部渲染tryExternalRenderer加载 /renderer/_framework/dcwriter5.js 和 jQuery，创建 WriterControlForWASM，再调用 LoadDocumentFromString(xml, 'xml')。
降级预览renderPreviewCanvas外部渲染不可用、加载失败或未生成 canvas 时，绘制结构化预览画布并显示可读 XML 文本。
编辑器适配createWriterControlAdapter统一封装加载、保存、命令执行、打印、打印预览，以及内容变化和选区变化监听。

五、命令、属性与未保存状态

所有菜单命令先进入 handleWorkbenchCommand。命令注册表会区分应用级命令、WriterControl 命令和占位命令。应用级命令由 runAppCommand 处理，编辑器命令由 runEditorCommand 经适配器发送给外部编辑器。

会标脏的典型行为
  • WriterControl 内容变化事件：onContentChanged 调用 session.markDirty()。
  • 执行可写编辑命令成功：runEditorCommand 调用 session.markDirty()。
  • 插入元数据输入域、片段模板、条码或二维码成功后标脏。
  • 当前文档绑定模板 id 时，同时调用 markTemplateDirty(templateId) 更新工作台模板状态。
辅助面板职责
  • 元数据面板支持选择、绑定到当前输入域、插入为输入域。
  • 片段模板面板支持选择片段，并将片段 XML 插入 WriterControl。
  • 属性面板通过 useElementInspector 读取或更新当前选中元素属性。
idle未加载文档或已清空。
dirty当前文档或模板存在未保存修改。
saving保存命令已触发，正在从编辑器取 XML 并请求接口。
saved保存成功，或模板保存接口失败后已降级写入工作台 mock 模板。
failed导入、渲染、校验、保存或编辑器能力缺失导致操作失败。

六、保存、导出、打印与模板上传

用户动作主调用链结果
保存saveToBackend → saveDocumentToBackend → adapter.saveXml → validateDocumentXml → POST /api/documents/save成功时清空校验问题并标记 saved。若真实保存接口失败且当前文档来自模板，则降级调用 saveTemplateContent 写入工作台 mock 模板。
下载 XMLdownloadCurrentXml → adapter.saveXml → downloadXml创建 XML Blob 并触发浏览器下载；若绑定模板，同时更新工作台模板内容。
另存为模板saveAsTemplate → 对话框取名 → saveTemplateAsContent 或 createTemplateFile生成新模板记录，切换当前会话到新模板，并刷新工作台数据。
打印/打印预览printDocument / openPrintPreview → adapter.print / adapter.openPrintPreview由 WriterControl 打印能力执行；失败消息进入状态提示。
上传模板requestTemplateUpload → beginTemplateUpload → completeTemplateUpload当前是前端内存 mock 上传队列，用定时器模拟待上传、上传中、已上传状态。

七、异常与降级路径

  • 切换文档前若 session.isDirty 为真，会弹出“未保存修改”确认，用户取消则停止切换。
  • 本地导入要求文件非空、不超过 10MB、文件名或内容像 XML，并且能被 DOMParser 正常解析。
  • 外部 DCWriter 资源加载失败、API 未暴露、未生成 canvas 时，自动回退到结构化 Canvas 预览。
  • 保存前会检查 XML 是否可解析，并校验必填输入域；必填项为空时保存失败并显示校验面板。
  • 真实后端保存失败但文档来源于模板时，会把 XML 写入工作台 mock 模板，并提示真实接口暂不可用。
  • 真实后端保存失败且文档是本地导入时，只会标记失败，不会伪装成模板保存成功。
